P13954 [ICPC 2023 Nanjing R] 红黑树

发表于 2026-07-21 本文字数： 594 阅读时长 ≈ 2 分钟

题意

给出一棵树和 a1,…,na_{1, \ldots ,n}a1,…,n​，优化：

fu,j=min⁡{[au=1]+∑vfv,i−1,[au=0]+∑vfv,i}f_{u,j}=\min\{ [a_u=1]+\sum_v f_{v,i-1},[a_u=0]+\sum_{v} f_{v,i} \} fu,j​=min{[au​=1]+v∑​fv,i−1​,[au​=0]+v∑​fv,i​}

思路

考虑发现这个函数有凸性，然后考虑证明。叶子结点的 [au=][a_u=][au​=] 函数是有凸性的，然后本质上是在对它们进行按位加然后取最小值还是具有凸性。

那么考虑答案，∑vfv,i−1\sum_v f_{v,i-1}∑v​fv,i−1​ 的图形可以看成是 ∑vfv,i\sum_v f_{v,i}∑v​fv,i​ 向右平移一格。然后考虑如果 [au=1][a_u=1][au​=1]，相当于将前者向上平移一格，可以看出是在零点的位置多加了一个斜率为 111 的位置，否则是在零点加斜率为 −1-1−1 的点。这样发现也可以证明是有凸性的。

那么答案就是 fu,0f_{u,0}fu,0​ 加上所有斜率为负数的点。即黑点个数加上所有斜率为负数的数量。

然后我们在做背包合并的时候可以懒插入一下，这样类似长链剖分复杂度。O(nlog⁡n)\mathcal{O}(n\log n)O(nlogn) 或 O(n)\mathcal{O}(n)O(n)。

思路

考虑有交的话按照最基本的一个题一个题讨论，考虑有这个题的所有人，那么只需要看这些人里是否有不为包含关系的人。考虑两个人讨论，即 Sy⊆SxS_y\subseteq S_xSy​⊆Sx​。那么有 ∣Sy∣≤∣Sx∣|S_y|\le|S_x|∣Sy​∣≤∣Sx​∣ 如果还有 Sz⊆SxS_z\subseteq S_xSz​⊆Sx​ 活着 Sz⊆SyS_z\subseteq S_ySz​⊆Sy​,那么它的大小只能小于 ∣Sy∣|S_y|∣Sy​∣ 或者在 [∣Sy∣,∣Sx∣][|S_y|,|S_x|][∣Sy​∣,∣Sx​∣] 了。那么这告诉我们只需要考虑和它大小相邻的集合是否有交即可。

P5311 [Ynoi2011] 成都七中

发表于 2026-07-20 本文字数： 507 阅读时长 ≈ 1 分钟

题意

给出一棵树，点有颜色。QQQ 次询问 l,r,xl,r,xl,r,x 在保留编号 [l,r][l,r][l,r] 的点的 xxx 点所在连通块大小。

思路

考虑如果一个点 ppp 和 xxx 联通，那么说明在 x↔px\leftrightarrow px↔p 的路径上全部都满足 u∈[l,r]u\in [l,r]u∈[l,r]。然后考虑先跳到 xxx 最顶上的点，这样我们只需要考虑这个点的子树即可，问题是子树大小 O(n)\mathcal{O}(n)O(n) 我们并不是很方便去统计。

考虑点分树，我们考虑点分树上 xxx 的祖先一定存在一个位置使得这个连通块的点都在这个祖先的子树里，因为跨子树就是相当于这个点是公共祖先，那么显然都在一个子树里。

我们考虑每一个结点存一下它点分树所有父亲到它路径上的极值和根。然后在每个根那里存自己节点的到根的结点范围和自己的颜色。

查询就变成了一个离线二位数点即可。

复杂度 O(∑Nlog⁡n+Qlog⁡n)=O(nlog⁡2n+qlog⁡n)\mathcal{O}(\sum N\log n+Q\log n)=\mathcal{O}(n\log^2n+q\log n)O(∑Nlogn+Qlogn)=O(nlog2n+qlogn)

即当 2sv>su2s_v>s_u2sv​>su​ 的时候移动到 vvv 更优，那么这样的 vvv 只有一个。然后我们就可以在点分树上算出每个节点的答案然后看真实树上相邻节点哪个最优然后移动到对应位置的点分树结点上即可。

P7274 草地

发表于 2026-07-18 本文字数： 1.1k 阅读时长 ≈ 3 分钟

题意

给出 n×mn\times mn×m 的黑白网格图，每一秒可以选择一个上下左右的方向，使得所有黑格向这个方向延伸一格，求最少几秒可以使所有黑格八连通。

n,m≤1000n,m\le 1000n,m≤1000

思路

考虑最后一步，如果选择上/左那么选择下/右一定也能连通，那么退回到上一个状态也是同理的能在同一秒进入一个“必胜态”。所以我们只需要考虑使用下/右操作即可。

考虑如果是 ttt 秒，使用了 aaa 个下操作，那么对于每一个黑点会覆盖成一个宽 aaa 长 t−at-at−a 的一个矩形。我们可以考虑对于任意两个黑点连一个 (∣xi−xj∣−1,∣yi−yj∣−1)=(pi,j,qi,j)(|x_i-x_j|-1,|y_i-y_j|-1)=(p_{i,j},q_{i,j})(∣xi​−xj​∣−1,∣yi​−yj​∣−1)=(pi,j​,qi,j​) 的边，那么我们相当于要求出一个 max⁡{p}+max⁡{q}\max \{ p \}+\max \{ q \}max{p}+max{q} 的最小生成树的权值。那么转化成了 LCT 板子题。考虑按 ppp 排序，然后对于目前维护 max⁡{p}\max \{ p \}max{p}，将这个边权以内的值加入 lct 维护的最小生成树，然后维护每一个时刻上面答案的最小值。

但是发现还是不行，因为黑点的个数 V=n⋅mV=n\cdot mV=n⋅m 的，所以边数是 V2V^2V2 时间复杂度 O(n2m2log⁡(nm))\mathcal{O}(n^2m^2\log(nm))O(n2m2log(nm)) 的比较坏。

那么为了考虑优化个东西，考虑是不是所有的边都是有用的。

考虑对于两点 u,vu,vu,v。我们发现，如果这两个点围成的矩形里面还有一个点 ccc，那么我们连边 u↔cu\leftrightarrow cu↔c 和 c↔vc\leftrightarrow vc↔v 一定更优秀。

不妨先考虑左上到右下的连边。如果说现在考虑到一个点 uuu，考虑左上角的所有点，对于一个最优的点，所有它的左上点也全都不需要有连边。那么我们可以拿单调栈维护。

那么对于同一行的相邻两个点 u1,u2u_1,u_2u1​,u2​，对于 u2u_2u2​ 来说只可能对行在 [xu1,xu2][x_{u_1},x_{u_2}][xu1​​,xu2​​] 的每一列最高的黑点连边，所以每一行的边数是 O(m)\mathcal{O}(m)O(m) 的，那么因为右上左下同理边的复杂度也就变成了 O(nm)\mathcal{O}(nm)O(nm)。

则最终复杂度变成了 O(nmlog⁡(nm))\mathcal{O}(nm\log(nm))O(nmlog(nm))。为了再优化一下，可以直接把已经联通的点都当成一个点来看，那么显然会省去不少的常数。

题意

给出 a1,…,na_{1, \ldots,n}a1,…,n​，QQQ 次操作每一次给出 l,r,xl,r,xl,r,x 将 ai∈{l,…,r}a_{i\in \{ l, \ldots,r \}}ai∈{l,…,r}​ 加上 xxx。求出每一次操作后将 aaa 拆分为若干个区间，定义价值为所有区间的极差之和，求最大价值。

n,q≤2×105,−108≤ai,x≤108n,q\le 2\times 10^5,-10^8\le a_i,x\le 10^8n,q≤2×105,−108≤ai​,x≤108

考虑每一个序列划分的区间都应该满足单调性，否则的话对答案没有贡献那么放到下一个区间一定不会使答案变得更劣。单调性考虑差分，那么也就是要求差分数组在一个区间里同号，那么对于选择的一个区间，我们得到的价值是 ∑i=l+1r∣di∣\sum_{i=l+1}^r \left| d_i\right|∑i=l+1r​∣di​∣。然后如果选择的区间是 [l,r],[r+1,q][l,r],[r+1,q][l,r],[r+1,q]，也就是说相当于刨掉了 dr+1d_{r+1}dr+1​ 这个点的贡献。

又有区间操作，于是我们考虑线段树维护。对于两个区间，我们考虑它是否刨掉左右两个端点 444 种状态分析求最大值。

对于两个同号的子区间不需要刨可以直接合并，剩下的就看是刨左区间的右端点好还是右区间的左端点好即可。

P5283 [十二省联考 2019] 异或粽子

发表于 2026-07-13 本文字数： 462 阅读时长 ≈ 1 分钟

题意

给出 nnn 个数，求所有区间 [l,r][l,r][l,r] 异或后的前 kkk 大值之和。

n≤5×105,k≤min⁡{n(n−1)2,2×105}n\le 5\times 10^5,k\le \min\left\{\frac{n(n-1)}{2},2\times 10^5\right\}n≤5×105,k≤min{2n(n−1)​,2×105}

思路

对于区间异或的问题，我们可以对序列做前缀异或来变成两点异或问题。然后我们发现，现在的问题是说我们要选择的两点需要有一个顺序关系。但是我们可以直接钦定随意选择两点，这样就变成了第 2k2k2k 大值，且因为两个相同的点异或为 000，所以不需要考虑两点不同。

之后我们把这 n+1n+1n+1 个值全部放进 01-trie 里，然后对于一般的 kkk 大值问题，考虑对于每一个点去查询当前的最大值，放进优先队列里，然后每一次访问就将下一个大值放进去。通过在字典树上二分解决。
