P7274 草地

发表于 2026-07-18 本文字数： 1.1k 阅读时长 ≈ 3 分钟

题意

给出 n×mn\times mn×m 的黑白网格图，每一秒可以选择一个上下左右的方向，使得所有黑格向这个方向延伸一格，求最少几秒可以使所有黑格八连通。

n,m≤1000n,m\le 1000n,m≤1000

思路

考虑最后一步，如果选择上/左那么选择下/右一定也能连通，那么退回到上一个状态也是同理的能在同一秒进入一个“必胜态”。所以我们只需要考虑使用下/右操作即可。

考虑如果是 ttt 秒，使用了 aaa 个下操作，那么对于每一个黑点会覆盖成一个宽 aaa 长 t−at-at−a 的一个矩形。我们可以考虑对于任意两个黑点连一个 (∣xi−xj∣−1,∣yi−yj∣−1)=(pi,j,qi,j)(|x_i-x_j|-1,|y_i-y_j|-1)=(p_{i,j},q_{i,j})(∣xi​−xj​∣−1,∣yi​−yj​∣−1)=(pi,j​,qi,j​) 的边，那么我们相当于要求出一个 max⁡{p}+max⁡{q}\max \{ p \}+\max \{ q \}max{p}+max{q} 的最小生成树的权值。那么转化成了 LCT 板子题。考虑按 ppp 排序，然后对于目前维护 max⁡{p}\max \{ p \}max{p}，将这个边权以内的值加入 lct 维护的最小生成树，然后维护每一个时刻上面答案的最小值。

但是发现还是不行，因为黑点的个数 V=n⋅mV=n\cdot mV=n⋅m 的，所以边数是 V2V^2V2 时间复杂度 O(n2m2log⁡(nm))\mathcal{O}(n^2m^2\log(nm))O(n2m2log(nm)) 的比较坏。

那么为了考虑优化个东西，考虑是不是所有的边都是有用的。

考虑对于两点 u,vu,vu,v。我们发现，如果这两个点围成的矩形里面还有一个点 ccc，那么我们连边 u↔cu\leftrightarrow cu↔c 和 c↔vc\leftrightarrow vc↔v 一定更优秀。

不妨先考虑左上到右下的连边。如果说现在考虑到一个点 uuu，考虑左上角的所有点，对于一个最优的点，所有它的左上点也全都不需要有连边。那么我们可以拿单调栈维护。

那么对于同一行的相邻两个点 u1,u2u_1,u_2u1​,u2​，对于 u2u_2u2​ 来说只可能对行在 [xu1,xu2][x_{u_1},x_{u_2}][xu1​​,xu2​​] 的每一列最高的黑点连边，所以每一行的边数是 O(m)\mathcal{O}(m)O(m) 的，那么因为右上左下同理边的复杂度也就变成了 O(nm)\mathcal{O}(nm)O(nm)。

则最终复杂度变成了 O(nmlog⁡(nm))\mathcal{O}(nm\log(nm))O(nmlog(nm))。为了再优化一下，可以直接把已经联通的点都当成一个点来看，那么显然会省去不少的常数。

对于两个同号的子区间不需要刨可以直接合并，剩下的就看是刨左区间的右端点好还是右区间的左端点好即可。

然后对于这种资源的转移和这种数据范围可以考虑网络流。对于所有的 di≥0d_i\ge 0di​≥0，则我们可以认为它最多做 did_idi​ 次减操作，目的是让其它的一些 dj<0d_j<0dj​<0 最终可以被转移成 ≥0\ge 0≥0 的状态。

所以我们将所有的 di≥0d_i\ge 0di​≥0 连 S→iS\to iS→i 的边，流量 did_idi​ 费用 000。然后将所有的 di<0d_i<0di​<0 连 i→Ti\to Ti→T，流量为 ∣di∣|d_i|∣di​∣ 费用为 000。

然后特别的，n+1n+1n+1 在差分中也有可能被访问，所以我们可以认为它是一个可以任意做的点，那么连 S→n+1S\to n+1S→n+1，流量为 +∞+\infty+∞ 即可。

P5283 [十二省联考 2019] 异或粽子

题意

给出 nnn 个数，求所有区间 [l,r][l,r][l,r] 异或后的前 kkk 大值之和。

n≤5×105,k≤min⁡{n(n−1)2,2×105}n\le 5\times 10^5,k\le \min\left\{\frac{n(n-1)}{2},2\times 10^5\right\}n≤5×105,k≤min{2n(n−1)​,2×105}

思路

对于区间异或的问题，我们可以对序列做前缀异或来变成两点异或问题。然后我们发现，现在的问题是说我们要选择的两点需要有一个顺序关系。但是我们可以直接钦定随意选择两点，这样就变成了第 2k2k2k 大值，且因为两个相同的点异或为 000，所以不需要考虑两点不同。

之后我们把这 n+1n+1n+1 个值全部放进 01-trie 里，然后对于一般的 kkk 大值问题，考虑对于每一个点去查询当前的最大值，放进优先队列里，然后每一次访问就将下一个大值放进去。通过在字典树上二分解决。
